git 本地结构

工作区：就是些代码的工作目录 git add 添加到暂存区

暂存区：临时存储 git commit 添加到本地库

本地库：包含所有的历史版本

​

git命令行操作

  1. git init 本地库初始化
  1. git status 状态查看
  1. **git add **提交到暂存区
  1. git rm –cached 从暂存区撤回
  1. git commit -m “commit message” 将暂存区内容提交到本地库

  1. git restore 撤销某个文件修改的操作 只能是工作区的文件

  2. **git reset HEAD **可以把暂存区的修改撤销掉(unstage),重新放回工作区

查看历史版本

git log

git log –pretty=oneline 每个历史只显示一行(hash值和日志)

git log –oneline 每个历史只显示一行且显示hash的部分值

git reflog 显示历史只显示一行,并且显示指针(要移动到版本多少步)

版本的前进后退

版本前进后退 实质就是指针的移动

  1. 基于索引值操作[推荐]

    git rest –hard [局部索引值]

  1. 使用 ^ 符号 （只能后退 不能前进）

    git reset –hard HEAD^^ ^ 表示后退的步数

  1. 使用 ~ 符号 只能后退

    git reset –hard HEAD~n 注：表示后退 n 步

git 分支操作

  1. git branch -v 查看所有分支

  2. git branch dev 创建新的分支

  3. git checkout dev 切换分支

  4. 合并分支

    新创建dev分支 修改test.text 然后在主分支上合并

    第一步:切换到接受修改的分支(被合并,增加新内容)上

    git checkout [**被合并分支名master]** 切换分支

    第二步：执行 merge 命令 (合并分支指令)

    *git merge [有新内容分支名**]**

git 远程库操作

  1. git remote add origin https://… 给远程库起别名
  1. git push origin master 推送

  1. git clone https://

    克隆仓库到本地

  2. git pull origin master

    在别人修改了分支里的内容时 你需要同步你的代码 使用pull命令

多人协同开发 push 可能会又冲突

如果不是基于 GitHub 远程库的最新版所做的修改，不能推送，必须先拉取。

拉取下来后如果进入冲突状态，则按照分支冲突解决操作解决即可。

将冲突的地方删了 保存为自己想要修改后的样子

然后再add 最后直接 git commit -m “modefied”
